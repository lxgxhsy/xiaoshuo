# 叙事系统解构

本文件只提炼结构机制，不复刻《诡秘之主》的角色、剧情、组织和专有设定。后续原创重写必须使用新的世界、新的人物、新的能力命名和新的因果链。

## 核心循环

参考文本最稳定的叙事循环不是“打怪升级”，而是：

1. 角色遭遇异常。
2. 异常暴露一条世界规则。
3. 角色只用自身视角解释异常。
4. 错误解释带来风险或代价。
5. 新信息修正主观视图。
6. 角色用有限资源做局部选择。
7. 选择触发更高层级的后果。

这个循环适合被实现为图谱：

- `KnowledgeNode` 记录异常背后的真相与危险等级。
- `PerspectiveView` 记录角色实际知道的版本。
- `ChapterEvent` 记录角色用什么信息、什么资源、什么目标做出行动。
- `ValidationReport` 检查角色是否越过了自身视角。

## 能力系统结构

有效的硬规则能力系统至少包含以下层：

- 路径：角色不能随意混搭能力方向。
- 阶位：高低差距必须影响冲突结果。
- 配方：能力获得需要知识来源。
- 材料：能力获得需要资源消耗。
- 仪式：能力获得需要条件和时间。
- 校准：能力需要长期行为与精神状态匹配。
- 失衡：错误使用会污染、失控或改变人格。
- 残留：强者死亡、物品失控或仪式失败会留下可追溯后果。

后续原创设定可以替换表层名词，但不能删除这些约束。

## 信息系统结构

信息在参考文本中同时是货币、武器和污染源。可抽象为：

- 普通知识：可安全传播。
- 职业知识：需要前置训练才能理解。
- 危险知识：知道本身会触发代价。
- 高层知识：低认知层只能得到变形版本。
- 不可靠知识：必须保留概率、怀疑和后续反证空间。
- 不可知知识：任何视角角色都不能完整掌握，只能留下阴影。

验证器要检查角色是否把不可靠信息当成绝对事实。

## 多视角结构

每个视角角色都应拥有独立故事线，而不是替主角传话。视角差异来自：

- 目标差异。
- 资源差异。
- 组织立场差异。
- 恐惧和欲望差异。
- 对同一知识节点的不同可靠性判断。
- 对高认知信息的不同衰减版本。

同一事件允许产生多个互相矛盾的版本，但每个版本必须在该角色视角内自洽。

## 卷结构模板

一卷可以按以下模式组织：

1. 异常引入：小事件暴露一条硬规则。
2. 视角扩展：至少两个角色从不同立场追踪同一异常。
3. 资源约束：角色为推进目标支付明确成本。
4. 错误认知：某条不可靠信息导致局部失败。
5. 规则兑现：危险知识、硬规则或失衡后果发生。
6. 真相分层：揭示一个中层真相，同时留下更高认知层阴影。
7. 代价结尾：角色得到推进，但失去资源、关系、身份或安全感。

每卷结尾不能只奖励主角，还必须改变世界图谱。
