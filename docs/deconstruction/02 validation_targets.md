# 可验证目标

本项目的重写稿必须先过结构检查，再进入正文润色。

## 图谱构建目标

- 每个主要角色都有 `core_desire`、`long_term_goal`、`current_goal`。
- 每个主要角色的客观状态和主观视图分离。
- 每个关键知识节点记录来源、可靠性、危险等级和认知层。
- 每个强力道具或能力记录副作用。
- 每次晋升、突破、学会新能力都有资源和时间消耗。
- 每个重要胜利都有因果链。
- 至少存在一个高于所有视角角色认知层的不可知节点。

## 章节生成目标

章节生成前必须先生成事件候选。事件候选至少包含：

- 视角角色。
- 行动角色。
- 行动目标。
- 使用的信息。
- 使用或考虑的资源。
- 消耗资源。
- 触发的硬规则。
- 得到的新信息。
- 代价和后果。

正文只能从通过验证的事件候选生成。

## 失败必须重写

以下问题不能降级成提示：

- 主角触发硬规则却免疫。
- 角色使用自己不知道的信息。
- 低阶角色常规击败高阶角色。
- 危险信息没有后果。
- 高认知层信息被低层角色完整理解。
- 资源或时间凭空出现。
- 反派因为剧情需要不使用可用资源。

出现这些问题时，生成系统必须重写事件或降低事件结果。
