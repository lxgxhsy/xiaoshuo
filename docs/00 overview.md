# 项目概览

本项目要做的是硬规则驱动的多视角小说世界模拟 / 生成系统。参考文本《诡秘之主》时，重点不是复刻剧情，而是抽取它能长期维持自洽感的工程化机制：能力有来源，信息有边界，知识有危险，角色只能基于自身视角行动，主角也必须承受同一套规则。

## 参考文本的结构性结论

### 1. 世界规则先于剧情爽点

非凡能力不是随意觉醒，而是由途径、序列、魔药、材料、扮演、消化、仪式和高层级代价共同约束。能力提升必须留下可追溯链条：

- 谁提供了配方或知识。
- 材料从哪里来。
- 使用了哪些资源。
- 消耗了多少时间。
- 是否满足晋升条件。
- 是否引入污染、失控、锚点、人性流失等后果。

生成系统不能让角色凭叙事需要突然拥有能力。任何能力变化都应该能映射为 `CONSUMES`、`COSTS`、`CAUSES`、`KNOWS`、`BELIEVES` 等图谱关系。

### 2. 信息本身是资源，也是危险源

参考文本中，许多关键推进不是靠战斗胜利，而是靠信息差、身份伪装、错误认知、隐秘组织交易和高层存在的信息压制。信息需要记录：

- 来源。
- 可靠性。
- 持有者。
- 传播路径。
- 危险等级。
- 所属认知层。
- 是否能被低认知层理解。
- 传播后是否衰减、扭曲或触发后果。

角色不能直接访问作者视角的真相。每个角色的决策必须从自己的 `PerspectiveView` 推出。

### 3. 多视角不是换镜头，而是换世界模型

同一事件在不同角色眼中应呈现不同意义。差异来自目标、立场、知识、恐惧、资源和认知层，而不是作者任意制造误会。

生成系统需要先构建客观图谱，再为每个角色投影出主观视图。章节草稿只能使用当前视角角色可感知、可理解、可误判的信息。

### 4. 主角优势来自因果链，不来自豁免权

主角可以赢，但胜利必须依赖可复查的条件：准备、信息差、环境、道具、第三方、代价、规则漏洞或对手目标限制。遇到高层级压制时，合理结果通常是逃脱、损失、被迫交易、暂时失败或付出长期代价。

系统必须能检测“主角因为是主角所以赢”的失败场景。

### 5. 高层世界保持不可知

完整世界不应被任何单一视角掌握。高认知层的信息对低层角色应表现为线索、象征、污染、梦境、误读、不可名状的压力或互相矛盾的传闻。

这对应 `docs/bdd.md` 的 F.4：图谱里必须存在高于所有视角角色认知层的知识节点。读者可以感到它存在，但角色不能直接拼出全貌。

## 生成流程目标

后续实现应遵循以下流程：

1. 构建客观世界图谱。
2. 构建角色主观视图。
3. 生成事件候选。
4. 检查资源、时间、信息、硬规则和认知层。
5. 对失败事件重写或降级。
6. 生成章节草稿。
7. 再次执行 BDD 校验。
8. 输出正文和结构化 `ValidationReport`。

任何正文生成都不应绕过图谱和校验层。
